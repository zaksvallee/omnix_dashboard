Design a complete revised UI/UX for a desktop-first security operations platform called ONYX. This is not a simple dashboard. It is a multi-surface command center used for live monitoring, tactical response, dispatch execution, governance/compliance, client communications, evidence review, and reporting. Keep every capability listed below. You may improve hierarchy, interaction design, density, readability, and visual polish, but do not remove workflow depth.

Primary design goal:
Create a premium, operationally credible, high-trust command platform that feels fast, decisive, and easy to scan under pressure. The UI should be modern, crisp, and information-dense without feeling messy. It must work especially well for operators who spend hours in it.

Design direction:
Desktop-first, with strong support for laptop and tablet landscape.
Serious, high-signal, operational look. No generic startup dashboard styling.
Clear hierarchy between urgent, active, degraded, blocked, review-needed, and complete states.
Strong typography, chips, badges, timelines, tables, structured cards, drawers, and panels.
Use color deliberately for operational meaning, not decoration.
Support light and dark concepts if helpful, but prioritize one polished primary theme first.
Focus on scanability, triage speed, and confidence.

Deliverables:
A complete Figma file with:
App shell and navigation system
Design system / component library
All primary pages below
Key modal / drawer / detail states
Empty / loading / error / degraded / offline states
Responsive desktop and laptop behavior
Interaction notes for major flows
A small clickable prototype for 3 critical flows:
Operations -> Tactical -> Dispatches -> Events -> Ledger
Admin -> Client Comms Audit -> exact lane drill-in
Live Operations -> Control Inbox -> draft refine / approve / reject

Interaction principles:
Operators should move fluidly between pages without losing context.
Drilldowns should preserve scope.
High-priority and degraded states must be visible from a distance.
Dense information should still feel calm and ordered.
Cross-surface continuity matters more than decorative novelty.
Do not make this look like a generic analytics dashboard or a consumer chat app.

Visual tone:
High-trust command center
Sophisticated but restrained
Sharp typography
Strong contrast
Clear color semantics
Subtle motion only where it reinforces hierarchy or workflow
No playful styling
No glassmorphism gimmicks
